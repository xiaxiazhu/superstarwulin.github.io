正则表达式收集

前言

作为IT老鸟，不能手快脚快的敲几个正则出来也是很不牛逼的，本文就是为了学习正则的一些过程以及常用的工具整理。

正则是什么：在编写处理字符串程序或者网页时候，经常会有查找符合某些规则的字符串的需要，正则表达式就是为了描述这些规则的工具。换句话说，正则表达式就是为了记录文本规则的代码。来自Jeffrey Friedl 的精通正则表达式的定义：正则表达式是强大、便捷、高效的文本处理工具。正则表达式本身，加上如同一门袖珍编程语言的通用模式表示法，富裕使用者描述和分析文本的能力。配合上特定工具提供的额外支持，正则表达式能够添加、删除、分离、叠加、插入和修正各类型的文本和数据

基本规则

完整的正则表达式由两种字符构成，特殊字符成为元字符，以及其他普通字符组成。元字符提供了更加强大的描述能力。

工具

egrep：很多操作系统-例如DOS，MacOS，Windows，Unix等有对应免费提供egrep。在指定了正则表达式以及需要检索的文件之后，egrep会蚕食用正则表达式匹配每个文件的每一行，并且显示能够匹配的行。
基本语法见 macworld

基础例子

匹配行-行的起始和结束

^cat$ ：以cat开始以及结束的行，没有空格等。

^&： 开始以及结束，无符号，空行

^： 开头，无意义，所有行都有开头

若干字符之一

<H[1-6]>与<H[123456]>完全一样，‘-’连字符，表示一个范围。

[0123456780abcdefABCDEF]和[0-9a-fA-F]是一样的。
还可以随心所欲的把字符范围和普通文本结合起来，例如[0-9A-Z._?!]。

排除型字符组

元字符^

[^...]不希望匹配...

[^1-6]不匹配1-6。

用点号匹配任意字符

元字符 .(通常称点好dot或者小点point)— 任意字符

03/19/76 、03-19-76、03.19.76麻烦的写法 03[-./]19[-./]76，简便的也可以写成03.19.76

多选结构-匹配任意子表达式

元字符 | — 或(or),依靠他，可以吧不同的子表达式组合成一个总的表达式，而这个总的表达式有能够匹配任意的子表达式。例如 bob|robert ，子表达式称为多选分支

gray|ey 意思是 gra 或ey；gr(a|e)y 也就是gray|grey;不能写成gr[a|e]y,这里|是和ae一样的普通字符。

多选表达式使用开始结束符号时候：
^From|Subject|Date: 匹配过程是 ^From 或者Subject或者Date:，几乎无实用性。

需要用括号来限制这些多选分支：
^(From|Subject|Date): ,匹配是： 1 起始From，然后：，2 起始subject，然后 ： 3 起始Date，然后：，这就可以*匹配邮件的开头信息。

忽略大小写

邮件头一般是忽略大小写的，一般的办法 [Ff] [Rr] [Oo] [Mm]取代From
egrep 的命令行 -i 可以忽略大小写匹配
例如： egrep -i “^(From|Subject|Date): ” mailbox

可选项元素

元字符 ？ —代表可选项，把它加在一个字符后面，就表示此处容许出现这个字符。

colou?r 来匹配 color或者colour；不匹配semicolon。

4th|4 等价于 4(th)?，括号内可以任意复杂。

重复出现

元字符 + 和*的作用于问好类似， +表示之前紧邻的元素出现一次或者多次；*表示之前紧邻的元素出现任意多次，或者不出现。
例如：
…+ 意思 匹配尽可能多的次数，如果一次匹配无法完成，则失败。
…* 意思 丕尽可能多次，如果实在无法匹配也不要紧。

？+ * 称为量词，因为它先顶了所作用元素的匹配次数。

重复出现次数的区间 …{min,max}，称为区间量词

例如： …{3,12}重复次数在3到12直接。 [a-zA-Z{1,5}]类匹配美国股票代码（1到5个字母）
问好对应区间量词{0,1}

设计模式微架构之 - 加载函器

加载器通用类

核心函数

/**
 *根据加载函数实现加载器
 *@param {Function} load 加载函数
 *@returns {Function} 加载器
 */
function createLoader(load) {
    var value, loading, handles = [], h;
    return function (handle, type) {
        //type默认为1
        //0:    不立即加载，不过加载完成之后执行回调函数
        //1:    立即加载，并在完成之后执行回调函数
        //2:    不立即加载，只在当前已经存在的情况下执行回调函数
        if (type !== 0 && !type) {
            type = 1;
        }
        if ((type & 1) && !loading) {
            loading = true;
            load(function (v) {
                value = v;
                while (h = handles.shift()) {
                    try {
                        h && h.apply(null, [value]);
                    }
                    catch (err) {
                        setTimeout(function () {
                            throw err;
                        }, 0)
                    }
                }
            })
        }
        if (value!==undefined) {
            handle && handle.apply(null, [value]);
            return value;
        }
        if (!(type & 2)) {
            handle && handles.push(handle);
        }//如果只在存在的情况下回调，则退出
        return value;
    }
}

优秀的特性以及代码解析

如果用这样的方式管理数据，或者管理远程的组件实例，就可以将管理对象集中起来，比如集中到一个文件道中，而调用实例，可以分布到各种各样的调用逻辑里面来执行。

执行器可以调用多次，加载器只发出一次网络请求，其他的从本地缓存获取。执行器如果没定义或者没执行，加载器不执行。

总结

对前端这样的富交互应用，大量的异步接口数据调用，上则提到加载器和调用器分离开来可以更好的维护代码以及分清模式，那一次加载，多次调用则让调用器更加广泛的应用到更多的逻辑中。

优秀特性中所提到的集中管理这样的微架构，将所有的数据集中起来管理，在大型网络富交互应用的前端部分可以让迭代开发更加容易，树形的模块结构显得更加可控，问题定位可以更快。

http协议详解

前言

作为IT老鸟，不能详细知道http这个整天玩弄的技术的基础，也是很不牛逼的，本文详细介绍学习http协议。

如果你是认真了，推荐一篇佳文拓宽视野：当你输入一个网址，会发生什么

URL详解

例子：

  • scheme: 制定底层使用的协议 （http,https,ftp）
  • host: HTTP服务器的ip地址或者域名
  • port#：HTTP服务器默认端口80，80可以省略，其他情况需要指明
  • path：访问资源的路径
  • query string：发送http服务器的数据
  • anchor： 锚

无状态协议

http是无状态的

同一个客户端的这次请求和上次请求是没有对应关系的，对http服务器来说，它并不知道两次请求来自同一个客户端。

解决：Cookie机制来维护状态

既然http是无状态，那 connection：keep-alive是怎么回事？

无状态是指协议对事物处理没有记忆能力，服务器不知道客户端是什么状态。从另一方面讲，打开一个服务器上的网页和你之前打开这个服务器的网页直接没有任何联系。

http消息结构

  1. request的三部分：Request line， http header ， body

    • request line ： 请求的总来，资源路径，协议版本
    • http header ： http 头部信息
    • body： 发送给服务器的query信息

      get方法时候，body为空（get只能读取服务器上信息，post写入body）

  1. response消息结构
    三部分：
    reques line ：协议版本、状态吗、message
    request header ： request 头部信息
    body： 返回资源请求的主体

get和post区别

http协议定义了很多与服务器交互的方法，最基本的有4种，分别是GET,POST,PUT,DELETE。 一个URL地址用于描述一个网络上的资源，而HTTP中的GET, POST, PUT, DELETE 就对应着对这个资源的查，改，增，删4个操作。 我们最常见的就是GET和POST了。GET一般用于获取/查询资源信息，而POST一般用于更新资源信息.

  • GET 提交的数据会放在URL之后，以?分割URL和传输数据，参数之间以&相连，如EditPosts.aspx?name=test1&id=123456。POST 方法是把提交的数据放在HTTP包的Body中。
  • GET 提交的数据大小有限制（因为浏览器对URL的长度有限制），而POST方法提交的数据没有限制.
  • GET 方式需要使用Request.QueryString 来取得变量的值，而POST方式通过Request.Form来获取变量的值。
  • GET 方式提交数据，会带来安全问题，比如一个登录页面，通过GET方式提交数据时，用户名和密码将出现在URL上，如果页面可以被缓存或者其他人可以访问这台机器，就可以从历史记录获得该用户的账号和密码.

状态码

Response 消息中的第一行叫做状态行，由HTTP协议版本号， 状态码， 状态消息 三部分组成。

状态码用来告诉HTTP客户端，HTTP服务器是否产生了预期的Response.

HTTP/1.1中定义了

5 类状态码。

状态码由三位数字组成，第一个数字定义了响应的类别

  • 1XX 提示信息 - 表示请求已被成功接收，继续处理
  • 2XX 成功 - 表示请求已被成功接收，理解，接受
  • 3XX 重定向 - 要完成请求必须进行更进一步的处理
  • 4XX 客户端错误 - 请求有语法错误或请求无法实现
  • 5XX 服务器端错误 - 服务器未能实现合法的请求

200 OK
请求被成功地完成，所请求的资源发送回客户端
302 Found
重定向，新的URL会在response中的Location中返回，浏览器将会使用新的URL发出新的Request
304 Not Modified
文档已经被缓存，直接从缓存调用
400 Bad Request
客户端请求与语法错误，不能被服务器所理解
403 Forbidden
服务器收到请求，但是拒绝提供服务
404 Not Found
请求资源不存在
500 Internal Server Error
服务器发生了不可预期的错误
503 Server Unavailable
服务器当前不能处理客户端的请求，一段时间后可能恢复正常
