web性能的极致优化

前人经验

web前端性能优化WPO，相信大多数前端同学或多或少都不会模式，在各自负责的业务中也有过实践。在业内，也有成熟的技术理论和大牛给出的法则供参考：

  1. Yahoo的web站点性能优化黄金法则
  2. 以及大名鼎鼎的优化大师Steve Souders

我们依葫芦画瓢，正常情况下，根据前人的经验，把性能优化的法则与我们的页面一一对照，可以有成倍的提升。
也即是：

小图片的合并，形成CSS Sprite，并优化图片
使用cdn,cdn域名差分
模块的异步加载    
图片的懒加载
设置缓存
CSS文件引用放在页面顶部，JS文件引用放在页面底部，并对代码压缩
缩小cookie体积
减少dns查找
…

众多优化中，效果最大的就是第一条规则，减少http请求的数量。图片的处理和css sprite的合并以及图片的懒加载。

立规则

  • 用规则规范大家的编码，保持性能持续的优良。
  • 性能优化不是前端同学自己的事情，需要业务各合作方的共同认同和支持。性能是最重要的业务功能点
  • 前端同学要增强自己的技术判断力，正确评估业务需求对性能的影响。同时要提升自身的沟通和影响力。

观点

用户感知和开发者关心的性能有差距

  • User Perception 用户感知：
    • 例子1： 首屏大图的快速渲染而其他具体信息懒加载给到用户的体验，相比于整体onload减少几秒要好很多
      • 例子2：在非常大加载量的页面中，纯粹的从比如10秒的加载减少到5秒，web qq中进度条过度页面出现会给用户良好的进度感知
  • network Time (TTLB) 网络加载数据
  • browser events (onload)
  • AFT Finish Render 首屏完全加载完毕
